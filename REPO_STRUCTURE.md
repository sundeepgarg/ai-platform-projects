# Suggested Showcase Repo Structure

If you create a separate LinkedIn-friendly repo, use this structure:

```text
enterprise-ai-platform-showcase/
|-- README.md
|-- architecture/
|   `-- flow-diagram.png
|-- docs/
|   |-- request-flow.md
|   |-- guardrails.md
|   `-- bedrock-mode.md
|-- screenshots/
|   |-- swagger-ui.png
|   |-- chat-response.png
|   `-- eval-output.png
|-- src/
|   `-- optional code or trimmed sample implementation
`-- linkedin/
    |-- post.md
    `-- featured-section-copy.md
```

## Recommendation

For personal branding, keep the showcase repo cleaner than the main learning repo:

- fewer branches
- cleaner README
- screenshots and diagrams
- architecture-first explanation
- clear learning outcomes
